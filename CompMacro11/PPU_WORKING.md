# РАБОЧИЙ механизм запуска кода на ПП (из Asteroids nzeemin)

ИСТОЧНИК: github.com/nzeemin/uknc-asteroids (ASTER1.MAC).
nzeemin — автор эмулятора UKNCBTL, код ТОЧНО работает под ним.
Это снимает наш блокер и объясняет, почему наш palette.mac не работал.

## ПОЧЕМУ НАШ КОД НЕ РАБОТАЛ
Мы загружали код на ПП по каналу 2 (правильно!) и запускали (команда 30),
НО наш ПП-код просто рисовал и возвращался. Прошивка ПП продолжала крутиться
в своём цикле (PC застревал в 174170) — мы не забрали управление.

В рабочем коде ПП-процедура СНАЧАЛА перенастраивает систему: запрещает
прерывания, бэкапит и чистит векторы, ставит СВОИ обработчики прерываний
(клавиатура, VSync), формирует списки отображения (display lists), и только
потом входит в свой главный цикл, слушая команды от ЦП.

## ЗАГРУЗКА КОДА НА ПП (канал 2) — РАБОЧАЯ ВЕРСИЯ

### Структура сообщения (Channel 2 data)
```
PPMSG:  .WORD  PPARR        ; указатель на массив параметров
        .WORD  177777       ; стоп-слово (НЕ 401! у нас было 401 — но тут 177777)
PPARR:  .BYTE  0            ; return value (0 = OK)
PPCMD:  .BYTE  1            ; команда (1 = allocate/выделить)
        .WORD  32           ; тип устройства (32 = память ПП)
PPAPP:  .WORD  0            ; аргумент (адрес ПП для операций с памятью)
PPACP:  .WORD  <PPEND-PPSTART>/2  ; CPU addr (для allocate = длина в словах)
PPLEN:  .WORD  <PPEND-PPSTART>/2  ; длина в словах
```

### Передача по каналу 2 (5 слов!)
```
PPSEN:  MOV  #PPMSG, R2
        MOV  #5, R3          ; 5 СЛОВ (не 5 байт!)
        BR   20$
10$:    MOVB (R2)+, @#176676 ; данные канала 2
20$:    TSTB @#176674        ; статус канала 2
        BPL  20$
        SOB  R3, 10$
        RETURN
```

### Последовательность запуска (RUNPPU)
```
RUNPPU:
        MOV  #177777, PPUCOM    ; init флаги
        CLR  PPUDAT
        CLR  PPUKEY
        CLR  FCanDraw
        CLR  FDrawDone
        CALL PPSEN              ; 1) ALLOCATE (команда уже =1 в PPCMD)
        MOV  PPAPP, PPUADR      ; запомнить выделенный адрес ПП
        MOVB #20, PPCMD         ; 2) команда COPY mem CPU->PPU
        MOV  #PPSTART, PPACP    ; начало кода ЦП
        CALL PPSEN
        MOVB #30, PPCMD         ; 3) команда RUN
        CALL PPSEN
        TST  PPUCOM             ; ждать подтверждения от ПП
        BNE  .-4
        RETURN
```

## ⭐⭐ КАК ЦП И ПП ОБЩАЮТСЯ — ЧЕРЕЗ ОБЩУЮ ПАМЯТЬ (планы)
Это подтверждает идею пользователя! ПП обращается к переменным ЦП через
свой регистр косвенного доступа 177010 (адрес/2):

ПП читает команду от ЦП:
```
        MOV  #<PPUCOM/2>, @#177010   ; адрес переменной PPUCOM в ОЗУ (делённый на 2)
        MOV  @#177014, R0            ; прочитать значение через косвенный доступ
```
ПП пишет в переменную ЦП:
```
        MOV  #<FCanDraw/2>, @#177010 ; адрес FCanDraw
        MOV  #1, @#177014            ; записать 1
```
Адрес ВСЕГДА делится на 2 (адресация словами). Регистры ПП:
177010 = адрес, 177012 = план 0, 177014 = планы 1&2.

## ПП-ПРОЦЕДУРА (PPSTART) — что делает по порядку
1. `MTPS #200` — запретить прерывания
2. Остановить дисковод: `CLR R0 / CALL @7132(R0)`
3. Забэкапить младшие адреса 0..1000 (векторы) в буфер BKP, очистить
   (кроме 272), поставить `MOV #2,(R3)` = RTI на адресе 0
4. Сохранить @#177054 (системный регистр)
5. Включить VSync ПП: `BIC #0400, @#177054`
6. Заглушить звук: `BIC #..., @#177716`
7. Поставить СВОИ обработчики:
   - клавиатура: адрес KEYINT в @#300, разрешить `BIS #100,@#177700`
   - VSync: адрес VSYINT в @#100, PSW в @#102
8. Сформировать ДВА списка отображения (display lists) — двойная буферизация
9. Очистить оба экрана (все 3 плана!)
10. Сигнал ЦП: можно рисовать (FCanDraw=1)
11. `MTPS #0` — разрешить прерывания
12. Войти в главный цикл PMAIN

## ГЛАВНЫЙ ЦИКЛ ПП (PMAIN) — диспетчер команд
```
PMAIN:  MOV  #<PPUCOM/2>, @#177010
        MOV  @#177014, R0       ; команда от ЦП
        BMI  PEXIT              ; <0 → выход
        ASL  R0
        ASL  R0                 ; *4
        ADD  R0, PC             ; переход по таблице
        BR   PMAIN              ; 0 — нет команды
        NOP
        BR   PDRSTR             ; 1 — рисовать строку
        NOP
        BR   PCLSTR             ; 2 — очистить строку
```

## ДВОЙНАЯ БУФЕРИЗАЦИЯ (8 цветов, экран)
- Два экрана: адреса 100000 и 134400
- Список отображения (display list) описывает строки экрана. Переключение
  экрана = смена значения по адресу 002346 (002350 / 004650).
- Палитра задаётся в display list (002340/002342) — слова палитры.
- ВАЖНО: ПП формирует display list и чистит ВСЕ 3 плана (177012 план0,
  177014 планы1&2) → отсюда 8 цветов.

## VSYNC обработчик (VSYINT)
Срабатывает по кадровому прерыванию, переключает экраны (двойная буфер.),
чистит теневой буфер, ставит FCanDraw для ЦП. Вектор @#100.

## КЛАВИАТУРА (KEYINT) — на ПП
Скан-коды читаются из @#177702, флаги клавиш пишутся в переменную PPUKEY
(косвенно через 177010). Коды: лево=116, право=133, верх=154, низ=134,
shift=105, стоп=4.

## СБОРКА (как у nzeemin)
```
macro11.exe ASTER1.MAC -l ASTER1.lst -o ASTER1.OBJ -rt11 -se -ysl 32
pclink11.exe -MAP ASTER1.OBJ -EXECUTE:ASTERA.SAV
```
MACRO11 (кросс-ассемблер) + PCLINK11 (линкер) → .SAV под RT-11.

## ⭐ ВЫВОД ДЛЯ НАШЕГО ПРОЕКТА
1. Загрузка кода на ПП через канал 2 — РАБОТАЕТ, но нужен ПОЛНЫЙ ПП-движок,
   который забирает управление (векторы, прерывания, display lists), а не
   просто рисует.
2. Спрайты в Asteroids рисует ЦП (через 176640/176642), ПП занят экранами/
   текстом/клавиатурой. Для 8 цветов ПП формирует display list со всеми
   3 планами.
3. Связь ЦП↔ПП — через общие переменные в ОЗУ (ПП читает их косвенно 177010).
4. У нас есть полный рабочий референс: work/asteroids_ref/ASTER1.MAC
   Можно изучать и адаптировать пошагово.
