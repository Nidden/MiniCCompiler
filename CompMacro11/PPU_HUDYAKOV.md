# Запуск кода на ПП и вывод спрайтов (по статье Худякова, ИНФО 1993 №1)

ИСТОЧНИК: Худяков Р.К. «Практика работы с периферийным процессором УКНЦ».
Это РЕШЕНИЕ главного вопроса — как загрузить и запустить наш код на ПП.

## 1. Загрузка и запуск кода на ПП — через массив параметров (МП), канал К2

ПП имеет системный механизм загрузки. Не нужны хитрости с прерываниями.
Передаём массив параметров (МП) по каналу К2 (регистры 176674/176676).

### Структура МП
```
MASSIV: .Byte 0      ; код ошибки или 0 (ответ ПП)
COMAND: .Byte 0      ; команда (см. ниже)
TYPDEV: .Word 32     ; устройство = ПП (десятичное 32 = окт 40? нет, тут 32 как есть)
ADR$PP: .Word 0      ; адрес в ОЗУ ПП (ПП сам вернёт при выделении)
ADR$CP: .Word 0      ; адрес в ОЗУ ЦП (откуда копировать)
LENGTH: .Word 0      ; длина программы в СЛОВАХ (не байтах!)
ADRMAS: .Word MASSIV ; всегда начальный адрес массива
        .Word 401    ; стоповый элемент
```

### Команды (поле COMAND)
- 01 = ВЫДЕЛИТЬ ПАМЯТЬ → ПП вернёт в ADR$PP адрес для программы
- 02 = ОСВОБОДИТЬ ПАМЯТЬ (нужны те же ADR$PP и LENGTH)
- 10 = ЧТЕНИЕ (из ОЗУ ПП в ОЗУ ЦП)
- 20 = ЗАПИСЬ (из ОЗУ ЦП в ОЗУ ПП)
- 30 = ПУСК (запуск с адреса ADR$PP)

### Последовательность запуска
1. COMAND=01 (выделить) → ПП вернёт ADR$PP
2. COMAND=20 (запись) → скопировать программу ЦП→ПП
3. COMAND=30 (пуск) → выполнить на ПП
4. COMAND=02 (освободить) → после работы

### Требования к программе для ПП
- Оформлена как подпрограмма, заканчивается RTS PC
- ПЕРЕМЕЩАЕМАЯ (адрес размещения заранее неизвестен!) — без абсолютных
  переходов внутри, адреса вычислять через PC.

### Подпрограмма передачи МП по К2 (рис. 2)
```
ZAGR:   .Enable LSB
        MOV   #ADRMAS, R0     ; адрес МП
        MOV   #5, R1
        BR    1$
2$:     MOVB  (R0)+, @#176676 ; передать байт по К2
1$:     TSTB  @#176674        ; ждать готовности канала
        BPL   .-4
        SOB   R1, 2$
        RTS   PC
```

### Готовый макрос-обёртка (рис. 4)
```
.MACRO $ZAGPP STADR, ENDADR
    JSR R5, ZAGPP
    .WORD <ENDADR-STADR/2>, STADR
.ENDM

; использование:
START:  $ZAGPP STARTPP, ENDPP   ; загрузить и запустить код ПП
        EMT 350                 ; выход в монитор
STARTPP: ... код для ПП (перемещаемый, RTS PC в конце) ...
ENDPP:
```
(полная подпрограмма ZAGPP с обработкой ошибок памяти — в статье, рис. 3)

## 2. Вывод спрайта на ПП (рабочий пример Худякова)

Регистры (всё в странице ввода-вывода ПП):
- 177016 — код цвета точки (0..7) = цвет спрайта
- 177024 — октет точек (8 бит = 8 точек, 1=зажечь)
- 177020/177022 — код цвета фона (planes), формат ниже
- 177026 — маска планов (запрет записи; НЕ блокирует красный/зелёный от ЦП)
- 177010 — регистр адреса в планах

### Процедура (спрайт 21 байт × 11 строк, белый)
```
        MOV #цвет_фона, @#177020    ; фон планы 1-4 точки
        MOV #цвет_фона, @#177022    ; фон планы 5-8 точки
        MOV #100000, @#177010       ; адрес в планах (левый верх)
        MOV #7, @#177016            ; цвет спрайта = 7 (белый)
        MOV #11, R0                 ; высота Y = 11
3$:     MOV #21, R1                 ; ширина X = 21 байт
2$:     MOVB (R2)+, @#177024        ; байт спрайта → экран (R2=адрес спрайта)
        INC @#177010                ; +1 адрес в планах
        SOB R1, 2$                  ; следующий байт строки
        ADD #<80.-21>, @#177010     ; перейти на следующую строку (80=ширина)
        SOB R0, 3$
        RTS PC
```

### ⭐ Режимы рисования (КЛЮЧЕВОЕ)
- **MOVB (R2)+, @#177024** — рисовать ПОВЕРХ фона (прозрачность, фон сохраняется)
- **MOV R3, @#177024** (через регистр, словно) — рисовать СО СТИРАНИЕМ фона
Для движения спрайта: рисуем MOVB, стираем восстановлением фона.

### Формат цвета фона 177020/177022
```
177020:  4    3    2    1   — номера точек в октете
        RGB  RGB  RGB  RGB  (красный,зелёный,голубой план)
177022:  8    7    6    5
        RGB  RGB  RGB  RGB
старший бит каждой тетрады не используется.
```
Пример: точка1 чёрная, 2 голубая, 3 зелёная... 8 белая →
177020 = 0b0011001000010000, 177022 = 0b0111011001010100

## 3. Карта памяти ПП (подтверждено)
- ОЗУ ПП: 0..77777 (32 КБ)
- ПЗУ ПП: 100000..177777 (32 КБ)
- Отладчик ПП: вход из меню установок, УПР+@ (как в мониторе ЦП)

## 4. Палитра / управление отображением (бонус)
Пары ячеек цвета по областям экрана:
- 2400,2402 — верхняя служебная строка
- 2470,2472 — сам экран
- 4700,4702 — меню установок
- 6750,6752 — нижняя служебная строка
Формат: 4 тетрады по слову (белый/жёлт/фиол/красн | бирюз/зел/голуб/чёрн),
каждая тетрада = Я R G B (яркость, красный, зелёный, голубой).
«Щёлканье» палитрой — мгновенная замена цветов, можно эффекты.

## 5. Звук (бонус, для игры) — таймер 177710/177716
Лучший способ — программируемый таймер в режиме прерываний (вектор 304).
Частота ноты = A/F, где A=62500 (период 16мкс), F=нужная частота Гц.
Таблица нот двух октав — в статье.

## ИТОГ: блокер СНЯТ
Теперь известен полный путь: ЦП загружает код в ПП через МП (канал К2),
запускает командой ПУСК. Код ПП использует спрайтовые регистры. Можем
писать минимальный тест: загрузить на ПП процедуру, рисующую один цветной
спрайт, и проверить.

## Следующий шаг (минимальный тест)
1. Написать перемещаемую ПП-процедуру: рисует один цветной октет/спрайт.
2. Обернуть в $ZAGPP, ЦП загружает и пускает.
3. Проверить на эмуляторе: появился ли спрайт нужного цвета.
Это первый реальный шаг к спрайтам Battle City.
