# UKNC_CODES.md — справочник кодов УКНЦ и MACRO-11

## ⚠️ КРИТИЧНО: восьмеричное vs десятичное

Это источник большинства багов в проекте. В MACRO-11:

| Запись      | Что это                    | Значение (дес) |
|-------------|----------------------------|----------------|
| `#33`       | восьмеричное (по умолчанию)| 27             |
| `#33.`      | десятичное (точка в конце) | 33             |
| `.BYTE 33`  | восьмеричное               | 27             |
| `.BYTE 33.` | десятичное                 | 33             |

**ПРАВИЛО:** в MACRO-11 число БЕЗ точки — восьмеричное, С точкой — десятичное.

**ПРОВЕРКА перед записью любого управляющего кода:**
1. Я хочу код N (в десятичном из таблицы ASCII)?
2. Если пишу `#N.` (с точкой) — N должно быть десятичным.
3. Если пишу `#N` (без точки) — N должно быть восьмеричным.
4. Перепутал → код уедет. ESC(27) написанный как `#33.` даст '!' (33).

## Управляющие коды (ASCII)

| Символ | Дес | Окт  | В MACRO-11 (дес) | В MACRO-11 (окт) | Назначение            |
|--------|-----|------|------------------|------------------|-----------------------|
| ESC    | 27  | 33   | `#27.`           | `#33`            | начало escape-послед. |
| CR     | 13  | 15   | `#13.`           | `#15`            | возврат каретки       |
| LF     | 10  | 12   | `#10.`           | `#12`            | перевод строки        |
| FF     | 12  | 14   | `#12.`           | `#14`            | очистка экрана+home   |
| BS     | 8   | 10   | `#8.`            | `#10`            | курсор влево          |
| TAB    | 9   | 11   | `#9.`            | `#11`            | табуляция             |
| '0'    | 48  | 60   | `#48.`           | `#60`            | цифра ноль            |
| 'Y'    | 89  | 131  | `#89.`           | `#131`           | буква Y               |
| ' '    | 32  | 40   | `#32.`           | `#40`            | пробел                |

**Перевод строки = CR + LF** (оба нужны: CR=13 возврат каретки, LF=10 строка вниз).

## УКНЦ escape-последовательности (консоль, порт @#177566)

ESC = 27 (дес). Параметры передаются ASCII-цифрами (значение + 48).

### Позиционирование курсора (gotoxy) — VT52-стиль
```
ESC 'Y' <col+32> <row+32>
27, 89, col+32, row+32
```
ВАЖНО: сначала COL (x), потом ROW (y). Координаты со смещением +32.

### Цвет текста
```
ESC 0240 <цифра>
27, 160, color+48
```
0240 окт = 160 дес. Цвет — ASCII-цифра '0'..'7'.

**Таблица цветов `setTextColor(код)` (выверено на эмуляторе):**

| Код | Цвет     | RGB-биты (б2=G б1=R б0=B) |
|-----|----------|---------------------------|
| 0   | чёрный   | 000                       |
| 1   | синий    | 001                       |
| 2   | красный  | 010                       |
| 3   | magenta  | 011 (красный+синий)       |
| 4   | зелёный  | 100                       |
| 5   | cyan     | 101 (зелёный+синий)       |
| 6   | жёлтый   | 110 (зелёный+красный)     |
| 7   | белый    | 111                       |

Раскладка — классический 3-битный цвет: бит0=синий, бит1=красный, бит2=зелёный.
Примечание: это палитра ТЕКСТА (консоль). Цвета ГРАФИКИ (cls/point/line, 0..3)
зависят от выбранной аппаратной палитры и могут отличаться.

### Цвет знакоместа (индекс 0)
```
ESC 0241 <цифра>   ; 27, 161, ...
```

### Цвет фона
```
ESC 0242 <цифра>   ; 27, 162, ...
```

### Режим экрана
```
ESC 0246 <цифра>   ; 27, 166, ...
'1' = 80 колонок, '2' = 40 колонок
```

## Видеопорты (графика)

| Порт       | Назначение                          |
|------------|-------------------------------------|
| @#176640   | адрес слова видеопамяти             |
| @#176642   | данные видеопамяти                  |
| @#177564   | статус консоли (TSTB, BPL = ждать)  |
| @#177566   | данные консоли (нечётный → MOVB)    |
| @#177546   | таймер LTC (~50 Гц)                 |

Экран: 320×264 (режим 0/2) или 640×264 (режим 1/3). Строка VRAM = 80 слов.
Бит 0 = ЛЕВЫЙ пиксель.

## Шаблон вывода байта в консоль

```asm
RTPUTC:
RTPUTC1: TSTB	@#177564    ; ждать готовности
        BPL	RTPUTC1
        MOVB	R0, @#177566 ; вывести (MOVB — порт нечётный)
        RTS	PC
```

## Чеклист при добавлении рантайм-функции с управляющими кодами

- [ ] Все `#N.` — десятичные, сверил по таблице ASCII
- [ ] ESC = `#27.` (НЕ `#33.`!)
- [ ] CR = `#13.`, LF = `#10.` (НЕ `#15.`/`#12.`!)
- [ ] Перевод строки = CR + LF, оба байта
- [ ] Вывод в порт через MOVB (порт 177566 нечётный)
- [ ] Перед MOVB — ожидание `TSTB @#177564 / BPL`
- [ ] Метки добавлены в проверку коллизий, если нужно
- [ ] Аргументы со стека: после `MOV R5,-(SP)/MOV SP,R5/MOV R0,-(SP)`
      первый аргумент = `4.(R5)`, второй = `6.(R5)`
