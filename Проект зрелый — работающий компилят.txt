Что уже есть — впечатляет:
Полноценный компилятор Mini-C → Macro-11 с лексером, парсером, кодогенератором, оптимизациями (peephole, inline, tail call, SOB, INC/DEC, свёртка констант). Рантайм для реального железа 1986 года. Редактор спрайтов. Стандартная библиотека с авто-подключением. Битовые операции, switch/case, do/while, print, строки.

Язык — чего не хватает:

Унарный минус — x = -y пока не работает как выражение, только 0 - y
Тернарный оператор a ? b : c — часто нужен для компактного кода
Глобальные переменные (не массивы) — int score; на уровне файла
Константы const int COLS = 7; — сейчас только #define нет
Указатели — хотя бы базовый уровень для работы с памятью напрямую
Функции возвращающие массив/указатель — нужно для match3 и других игр

Рантайм — что реально нужно:

printnum(n) — вывод числа на терминал (нет ни одного способа вывести переменную)
getTimer() — читать LTC @#177546 → счётчик кадров/времени, нужен для анимаций
Горизонтальный scroll — сдвиг видеопамяти, УКНЦ аппаратно поддерживает
cls() без аргумента — сейчас cls(1) обязателен, неудобно

Инструменты:

Отображение размера кода в статусной строке — сколько байт занимает программа
Экспорт .mac одной кнопкой в файл на диске
Нумерация строк в правой панели (asm)
Поиск/замена в редакторе кода
История отмены (Ctrl+Z) в редакторе

Редактор спрайтов:

Flood fill (заливка) — самый нужный инструмент которого нет
Копировать/вставить область
Зеркальное отражение по горизонтали/вертикали
Анимация — несколько кадров для одного спрайта

Самое ценное из незаделанного — printnum(n). Без него невозможно отлаживать игровую логику — нельзя вывести счёт, координаты, состояние. Это можно реализовать как функцию StdLib:
cvoid printnum(int n) {
    if (n < 0) { print("-"); n = 0 - n; }
    // рекурсивный вывод цифр...
}
Но рекурсия требует стека — лучше итеративно через массив цифр. Реализуемо на Mini-C уже сейчас.
